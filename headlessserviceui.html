Headless Service UI Example

Description

The HeadlessServiceUI example demonstrates how to create the UI counterpart for a headless app (in this case, HeadlessService. Make note that this is NOT a standalone, independently deployable application, it is packaged with HeadlessService sample. Hence, you must have both project folders in the same workspace folder to keep the project references valid. To build the projects, please build HeadlessService first and then HeadlessServiceUI. The combined bar package will be in the HeadlessServiceUI project.

Overview

The HeadlessService and the HeadlessServiceUI project together demonstrates the following

  • How to create a long running headless app i.e. once launched, it can run for as long as it wants.
  • How to communicate between the headless portion and the UI portion of the app via a file stored in the filesystem. (This is extremely useful if you have your UI portion updating a settings file that the headless portion needs to sync with immediately or vice versa.)
  • How to launch a headless service on device startup and on install.

HeadlessService is a very simple long running headless app that simply flashes the LED 20 times. Its UI counterpart, the HeadlessServiceUI, lets the user send a signal to the HeadlessService to restart the LED flashing. The communication is facilitated through read/write of a file via QSettings.
